Wire protocol

All messages are length-prefixed TCP frames with a fixed 24-byte header.

Frame format

binary
[ MsgHeader (24 bytes) ][ payload (variable) ]

MsgHeader:
  magic       uint32   0xD157C0DE
  version     uint32   1
  msg_type    uint16   MsgType enum
  flags       uint16   reserved
  payload_len uint32   bytes after header
  seq         uint64   per-connection sequence number

All multi-byte integers are little-endian. A peer that reads a frame with a non-matching magic or an unknown version closes the connection.

Ports

PortPurpose
7700Coordinator control — NODE_JOIN, HEARTBEAT, LAYER_ASSIGN
7701Data plane — TENSOR_FORWARD (node → node)
7702API plane — INFER_REQUEST / INFER_TOKEN / INFER_DONE
7703VM control — tensor alloc/write/read, op dispatch, collectives, checkpoints
8080HTTP(S) — dashboard, WebSocket, OpenAI-compat (dist-server)

Connection lifecycle (a rig joining a pool)

  1. Installer execs dist-node --pair distpool://….
  2. dist-node opens a WebSocket to wss://<apex>/ws/agent.
  3. Sends { "type": "pair", "token": "…", "hostname": "…", "gpus": […] }.
  4. Server validates the token (single-use, TTL 5 min), inserts a rigs row, mints an agent_key.
  5. Server replies { "type": "paired", "rig_id": N, "agent_key": "…" }.
  6. dist-node persists the agent_key and keeps the socket open.
  7. Subsequent reconnects: { "type": "auth", "agent_key": "…" }.

Inference request (OpenAI-compat)

The client hits the control plane with a standard OpenAI POST:

http
POST https://alice-alpha.<apex>/v1/chat/completions
Authorization: Bearer sk-dist-<48 hex>
Content-Type: application/json

{
  "model": "llama-3-70b",
  "messages": [{ "role": "user", "content": "Hello!" }],
  "stream": true
}

Server picks an online rig in the pool, forwards the request over its WebSocket, and streams the response back as SSE (OpenAI chunk format + [DONE]) or buffered JSON.

Data plane (node → node)

Once inference starts, each node sends TENSOR_FORWARD frames to the next node's TCP :7701. The payload is raw bytes of the activation tensor plus a small header describing dtype and shape.

There's no encryption on the data plane — assume a trusted LAN or terminate TLS upstream (Caddy, Cloudflare Tunnel, Tailscale).

Auth

  • Browser sessions: GitHub OAuth round-trip in prod; -dev flag exposes POST /auth/dev for local work. Session cookie = HMAC-signed user id.
  • Rig agents: 5-minute single-use pair tokens → permanent agent_key stored on the rig.
  • API keys: SHA256-hashed at rest, plaintext returned exactly once on creation, 12-char prefix shown in the UI for identification.
