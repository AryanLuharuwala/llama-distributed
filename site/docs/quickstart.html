Quickstart

Build the binaries and run a distributed inference test in under 10 minutes.

1. Clone the repo

A single --recursive clone is all that's required. llama.cpp is a git submodule, so you don't need a separate checkout.

shell
git clone --recursive https://github.com/AryanLuharuwala/llama-distributed
cd llama-distributed

2. Build

The build script auto-detects your toolchain. Pick an accelerator flag:

shell
# CPU build (takes ~5 min for llama.cpp)
./scripts/build.sh

# CUDA (NVIDIA)
./scripts/build.sh --cuda

# Metal (Apple Silicon)
./scripts/build.sh --metal

# Vulkan (AMD / Intel)
./scripts/build.sh --vulkan

Binaries land in ./build/.

No separate llama.cpp checkout needed. It ships as a git submodule at third_party/llama.cpp.

3. Single-machine test

Three processes on localhost — coordinator, two nodes, client.

4 terminals
# Terminal 1 — coordinator
./build/dist-coordinator --model model.gguf --min-nodes 2

# Terminal 2 — node 0
./build/dist-node --server 127.0.0.1 --data-port 7701

# Terminal 3 — node 1
./build/dist-node --server 127.0.0.1 --data-port 7704

# Terminal 4 — client
./build/dist-client --server 127.0.0.1 --prompt "Hello!"

Tokens stream to stdout as they're generated.

4. Run on a real cluster

Same binaries, different IPs. One coordinator, N node agents.

shell
# On the coordinator host
./build/dist-coordinator \
  --model /models/llama-3-70b.gguf \
  --min-nodes 3

# On each GPU server
./build/dist-node --server <coordinator-ip>

# From anywhere
./build/dist-client \
  --server <coordinator-ip> \
  --prompt "Explain ring-AllReduce in one paragraph." \
  --max-tokens 256

5. VM / unified-computer mode (optional)

VM mode adds distributed tensor memory, collective ops, and checkpointing on top of the base pipeline. Pass -DDIST_BUILD_VM=ON to CMake to enable (currently gated off in CI).

shell
# Coordinator with VM layer
./build/dist-vm-coordinator \
  --model  /models/llama-3-70b.gguf \
  --min-nodes 3

# Node agents with VM layer (same flags as dist-node)
./build/dist-vm-node --server <coordinator-ip>

Client API in C++:

cpp
VmContext ctx({"coordinator-ip"});
ctx.infer(token_ids, 256, [](int32_t tok, bool last) {
    std::cout << tok;
});
ctx.checkpoint().wait();   // snapshot entire cluster state
