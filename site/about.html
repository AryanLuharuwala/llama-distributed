About

An open-source project for pooling your GPUs.

llama-distributed is an open-source system for running large language models across a heterogeneous pool of GPU machines. Apache 2.0, community-developed, maintained on GitHub.

What we're building

Three things, in descending order of maturity:

  1. A layer-parallel inference pipeline in C++ on top of llama.cpp — production-ready for single-pool deployments.
  2. A Go control plane that does OAuth, pool routing, API key management, and exposes OpenAI-compat endpoints — stable.
  3. An experimental VM layer adding distributed tensor memory, collective ops (ring-AllReduce, AllGather), and checkpointing — not production-ready, currently gated off in CI.

Principles

  • Works with what you have. Mix CUDA, Metal, Vulkan, and CPU rigs. Don't assume NVLink, InfiniBand, or homogeneous hardware.
  • OpenAI SDK drop-in. If we change the API, we've failed. Users shouldn't know they're talking to a distributed system.
  • Zero-friction onboarding. One command to install, one click to pair, one copy-paste for the OpenAI snippet. Anything longer is a bug.
  • Pure library usage of llama.cpp. We don't fork or patch upstream. Updates are one git pull.
  • Plain HTML for this site. No framework, no bundler, no 200 MB of node_modules. Open the file, it works.
