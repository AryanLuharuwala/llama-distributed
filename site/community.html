Contribute

How to send a patch.

All development happens on GitHub. The short version:

  1. Find an issue labeled good first issue on the issues page, or open one describing what you want to change.
  2. Fork the repo, branch, make the change.
  3. Run ./scripts/build.sh and the local test suite (ctest --test-dir build).
  4. Open a PR against main. Reference the issue in the description.
  5. CI runs the 5-target build matrix. If green, we'll review within a few days.

Priorities right now, in order of "someone should take these":

  • Partial model loading — use tensor_buft_overrides to skip loading the full GGUF on every node.
  • Python bindings for VmContext — a pyo3 or nanobind wrapper, good first issue for C++/Python hybrid devs.
  • TURN / STUN support — for rigs behind NAT.
  • Dashboard polish — the embedded SPA is still minimal. React or vanilla JS improvements welcome.

Code style: we follow Google C++ style for the C++ side and Google Go style for the Go side. clang-format and gofmt run in CI.
